# 업데이트 이력

ACCESS LINK 앱의 큰 작업 단위 변경 내용을 날짜별로 누적 기록한다.

## 2026-07-15

### 첫 화면 재구성

- 사용자 첫 화면을 안면인식 / 카드 리더기 중심의 대기 화면으로 변경했다.
- 하단 탭을 제거하고 관리자 진입 버튼을 우측 상단의 작은 버튼으로 분리했다.
- 어두운 파랑과 흰색 계열을 섞은 사이버틱한 화면 톤을 적용했다.
- 카드 리더기 영역에는 연결 상태, 최근 인증 기록, 수동 개방 버튼을 표시한다.

### 안면 인증 문 개방 설계 정정

- Android OS 생체 인증 프롬프트는 지문이 함께 노출될 수 있어 안면 전용 출입 인증 요구와 맞지 않는 것으로 정정했다.
- AndroidX Biometric 의존성과 OS 생체 인증 호출은 제거했다.
- 안면 인증 성공 후 문 개방 정책 함수는 전용 얼굴 비교 엔진 연결 지점으로 유지했다.
- 현재 안면인식 버튼은 전용 얼굴 비교 모델이 필요하다는 로그를 남기며, 얼굴 검출만으로 문을 열지 않는다.
- 향후 구현은 전면 카메라 기반 얼굴 등록, 등록 얼굴 템플릿 저장, 실시간 얼굴 비교, 성공/실패 UI 표시, 성공 시 문 개방 순서로 진행한다.

### 전면 카메라 얼굴 검출/등록 UI 추가

- CameraX와 ML Kit Face Detection 의존성을 추가했다.
- Android 카메라 권한을 추가했다.
- 사용자 첫 화면의 안면인식 영역에 전면 카메라 프리뷰와 얼굴 검출 상태를 표시한다.
- 관리자 화면에 `얼굴` 탭을 추가했다.
- 관리자 얼굴 등록 화면에서 얼굴이 1명 감지될 때 등록자 이름을 저장할 수 있게 했다.
- 등록 얼굴 목록을 SharedPreferences에 저장/복원한다.
- 현재 단계는 얼굴 검출과 등록 UI까지이며, 동일인 판별과 문 개방은 얼굴 비교 모델 연결 전까지 막는다.

### 온디바이스 얼굴 비교 및 문 개방 연결

- Apache-2.0 라이선스의 FaceNet TFLite 모델(`facenet.tflite`)을 앱 assets에 추가했다.
- TensorFlow Lite 런타임 의존성을 추가했다.
- 카메라 프레임에서 얼굴 영역을 잘라 160x160 RGB 입력으로 변환하고 128차원 얼굴 임베딩을 생성한다.
- 관리자 얼굴 등록 시 이름과 얼굴 임베딩을 함께 저장하도록 변경했다.
- 사용자 첫 화면에서 등록 임베딩과 현재 얼굴 임베딩의 L2 거리를 비교한다.
- 등록 얼굴과 일치하면 초록 상태를 표시하고 문 개방 명령을 전송한다.
- 미등록 얼굴이면 빨간 상태로 `등록되지 않은 사용자입니다`를 표시하고 문을 열지 않는다.
- 반복 개방과 반복 실패 로그를 막기 위해 안면 개방 쿨다운과 실패 로그 간격을 적용했다.

### 관리자 UX 재구성 및 릴레이 진단 추가

- 관리자 화면을 단순 탭 나열에서 대시보드 중심 구조로 변경했다.
- 관리자 첫 화면에 장비 연결, 문 설정, 카드/얼굴 등록 상태를 요약 표시한다.
- `설치 설정 시작` 흐름을 추가해 장비 연결, 릴레이 테스트, 문 이름 지정, 인증 수단 등록, 자동 개방 대상 지정 순서로 작업할 수 있게 했다.
- 문 설정 화면에 “장비 1대 = Relay 0/1 두 개” 기준의 현장 배선 매핑 설명을 추가했다.
- 장비 진단 화면에 관리자 전용 릴레이 진단 패널을 추가했다.
- 릴레이 진단에서는 Relay 0, Relay 1, 전체 Relay를 선택해 지정 시간 ON, 지속 ON, OFF 명령을 송신할 수 있다.
- 릴레이 진단 UI는 실제 릴레이 상태를 단정하지 않고 마지막 송신 패킷과 송신 로그만 표시한다.

### 실물 설치 화면 참고 기반 사용자 화면 재구성

- 실물 태블릿 설치 사진을 기준으로 상단 브랜드·시간·장비 연결 상태·관리자 진입 영역을 재배치했다.
- 중앙 안면인식 영역을 세로형 카메라 프리뷰로 확대하고 얼굴 가이드, 얼굴 인증 안내, 인증 상태 색상을 적용했다.
- 하단 카드 영역을 RFID 배지와 카드 태그 안내 중심으로 변경했다.
- 기존 문 OPEN 버튼, 최근 인증 기록, 얼굴 인증 및 카드 인증 연결은 유지했다.
- 릴레이 실제 상태를 단정하는 표시 없이 인증 성공 시 문 개방 명령 전송 안내만 표시한다.

### 관리자 화면 사용 흐름 개선

- 관리자 첫 화면을 기능 목록 중심에서 설치 순서 중심으로 변경했다.
- `장비 연결 → 문·릴레이 설정 → 사용자 등록 → 인증 후 열 문 선택`의 4단계 상태를 한눈에 표시한다.
- 문·릴레이, 카드 등록, 얼굴 등록, 자동 개방 설정을 한국어 설명과 함께 분리했다.
- Wiegand·RS-232·RS-485는 `고급 장비 진단`으로 묶어 일반 설치 흐름과 분리했다.
- 장비 진단 화면의 연결 속도, 네트워크 링크, 카드 리더기 데이터 설명을 현장 작업자 기준 문구로 변경했다.
- 장비 진단 화면은 USB 연결·권한·릴레이 테스트를 기본으로 보여주고, Wiegand·RS-232·RS-485는 접을 수 있는 고급 통신 진단으로 분리했다.
- 장비 진단을 요약 홈, 장비 연결, 릴레이 테스트, 카드 리더기 데이터, RS-232, RS-485 상세 페이지로 나눴다.

### 설치 마법사 및 관리자 진단 흐름 개선

- 실제 출입통제 관리자 앱의 설치 흐름을 참고해 관리자 작업을 단계별 마법사 형태로 정리했다.
- 설치 마법사에서 현재 진행 단계, 완료 단계, 다음 작업을 표시한다.
- 사용자 등록 단계에서 카드 등록과 얼굴 등록을 각각 바로 선택할 수 있게 했다.
- 관리자 하위 화면 이동 시 이전 화면으로 돌아오는 경로를 유지하도록 화면 이동 스택을 추가했다.

## 2026-07-20

### 코드 리뷰 반영 및 운영 UX 보완

- 백그라운드 복귀 후 관리자 화면이 PIN 없이 유지되지 않도록 관리자 라우팅과 컨트롤러 권한 검사를 보강했다.
- 얼굴 인증은 여러 프레임에서 같은 얼굴이 연속 확인될 때만 개방 명령을 전송하도록 변경했다.
- 얼굴 인증 결과와 릴레이 송신 결과를 분리하고, 대상 문 없음·송신 실패를 실패 상태로 표시하도록 수정했다.
- 등록 카드에 자동 개방 대상 문이 없을 때 인증 성공으로만 기록되지 않도록 별도 오류 기록을 추가했다.
- 출입 이벤트를 영속화하고 관리자 기록 화면에서 필터링해 확인할 수 있게 했다.
- 설정·얼굴 특징값·출입 기록 저장값을 Android Keystore 기반으로 암호화하고 앱 백업 대상에서 제외했다.
- 카메라 분석기와 CameraX 생명주기 해제를 보강하고 정적 분석 오류를 수정했다.
- USB 연결 상태, 이더넷 물리 링크, 카드·Wiegand·RS-232·RS-485 진단 화면의 안내 문구를 구분했다.
- 관리자 자동 개방 설정에서 카드 자동 개방과 얼굴 인증 대상 문을 구분해 설명했다.
- 수동 문 개방의 짧은 시간 내 중복 송신을 제한했다.

## 2026-07-23

### ACCESS LINK 장비 이해 및 앱 방향 문서 추가

- 사내 마케팅 자료와 프로토콜 문서 기준으로 ACCESS LINK의 제품 포지션을 정리했다.
- 출입통제는 대표 사용 사례이며, 장비 자체는 앱·서버·태블릿과 현장 장비를 연결하는 범용 인터페이스 장비로 정의했다.
- USB-C와 Ethernet 연결 역할, Relay/Wiegand/RS-232/RS-485/Digital Input 기능, 앱에 필요한 운영·관리·고급 진단 영역을 분리해 정리했다.
- 향후 개발 기준으로 문 중심 표현보다 Relay/Input/Wiegand 같은 장비 기능 중심 용어를 우선 사용하도록 정리했다.
- Ethernet 실시간 수신 구현 전 제조 담당자에게 확인해야 할 IP, 포트, TCP/UDP/HTTP 방식, 이벤트 수신 방식 질문을 문서화했다.

### 관리자 화면 릴레이 용어 정리

- 관리자 기본 메뉴의 `문·릴레이 설정`을 `릴레이 설정`으로 변경했다.
- 자동 개방 설정 문구를 `인증 출력 설정`으로 바꿔 출입문 전용 표현을 줄였다.
- Relay 0/1 설정 화면에서 표시 이름, 사용 여부, 출력 시간 중심으로 설명을 변경했다.
- 사용자 화면의 수동 버튼 안내를 `문 개방`에서 `수동 릴레이 출력`으로 변경했다.
- 릴레이 진단 화면에서 Relay 0+1 동시 출력 선택 버튼을 숨겨 실수로 두 출력을 동시에 실행하지 않도록 했다.

### 관리자 홈 정보 구조 개편

- 관리자 홈을 설치 순서 중심에서 장비 기능 중심 구조로 변경했다.
- 홈 섹션을 `장비 기본 설정`, `인증과 동작`, `설치 보조`로 나눴다.
- `장비 연결`, `출력 설정 (Relay)`, `입력 설정 (Input)`, `통신 진단`, `카드 등록`, `얼굴 등록`, `인증 동작 설정`, `기록` 카드로 주요 작업을 바로 찾을 수 있게 했다.
- 처음 설치 순서는 관리자 홈의 보조 동선으로 낮춰, 운영 관리자와 설치자가 필요한 기능을 먼저 찾을 수 있게 했다.
### 관리자 Input 화면 분리

- 관리자 홈의 `입력 설정 (Input)` 메뉴를 장비 진단 화면과 분리했다.
- 별도 화면에서 Input 0/1의 최근 수신 상태와 마지막 원문 패킷을 확인할 수 있게 했다.
- 입력 신호 기반 Relay 자동 실행은 아직 구현하지 않았으며, 다음 단계 정책 설정으로 분리하도록 안내 문구를 추가했다.
- Wiegand, RS-232, RS-485 상세 확인은 기존 고급 장비 진단 화면으로 이동하도록 연결했다.
